# v0.05 自动保存 + localStorage 结构整理

## 概述

v0.05 的两个核心目标：
1. **整理 localStorage 结构** — 将散落的 key 按三层分类合并到统一结构下
2. **实现自动保存** — 防抖 + 定时兜底双重机制保存用户章节数据，支持启动恢复

---

## 一、localStorage 结构整理

### 当前散落的 key

```
storyeditor_settings          ← 编辑器配置（主题、字号）
storyeditor_labels            ← 字段别名
storyeditor_chapter_cols      ← 章节视图列配置
storyeditor_templates         ← 用户模板
storyeditor_deleted_templates ← 已删除的默认模板
storyeditor_template_keys     ← 模板键名
storyeditor_structs           ← 结构类型定义
```

### 目标结构（三层分离）

```json
{
  // ━━━━━ 第 1 层：编辑器自身配置 ━━━━━

  "storyeditor_config": {
    "meta": {
      "version": 1,
      "updatedAt": "2026-07-01 14:30:00"
    },
    "theme": "dark",
    "fontSize": 14,
    "labelColorMode": "theme",
    "labels": {
      "speaker": "角色",
      "headimage": "立绘"
    },
    "chapterCols": {
      "speaker": true,
      "text.zh": true
    }
  },

  // ━━━━━ 第 2 层：数据结构定义（模板 + 结构类型） ━━━━━

  "storyeditor_schema": {
    "meta": {
      "version": 1,
      "updatedAt": "2026-07-01 14:30:00"
    },
    "templates": { /* 用户自定义模板字段 */ },
    "deletedTemplates": [ "meta.def1" ],
    "templateKeys": { /* 模板键名自增 */ },
    "structs": [ /* 结构类型定义 */ ]
  },

  // ━━━━━ 第 3 层：用户当前编辑的 JSON 文档（v0.05 新增） ━━━━━

  "storyeditor_document": {
    "meta": {
      "version": 1,
      "fileName": "chapter_01.json",
      "savedAt": 1721234567890,
      "displayTime": "2026-07-01 14:30:00"
    },
    "data": { /* 用户 JSON 原始内容 */ }
  }
}
```

### 迁移策略

用户浏览器中已有旧 key 数据，需在启动时执行一次迁移：

1. **读取旧 key** → 写入新结构对应位置 → **删除旧 key**
2. 迁移仅在检测到旧 key 存在时执行一次，后续直接读新结构
3. 提供标记 `_migrated_v05` 作为迁移完成的标志（可选方案：检测旧 key 不存在即代表已迁移）
4. 导出的配置格式也同步更新

## 二、自动保存功能

### 数据范围

仅保存 **第 3 层：用户正在编辑的章节 JSON 数据**（含文件名等元信息）。第 1、2 层已有各自的即时保存机制，不纳入自动保存。

### 存储设计

| Key | 内容 |
|-----|------|
| `storyeditor_document.data` | 当前章节 JSON 完整数据（`store.chapter` 的深拷贝） |
| `storyeditor_document.meta` | 元信息 `{ version, fileName, savedAt, displayTime }` |

### 触发机制

采用**变化防抖 + 定时兜底**双重机制：

```
用户操作（编辑 + UI 交互）
    │
    ├─→ 重置防抖定时器 (10s)
    │      └─ 10s 内无新操作 → 检查变更 → 有则保存
    │
    └─→ 兜底定时器 (60s 周期)
           └─ 每次触发 → 检查数据版本（_dataVersion）是否有变更 → 有则保存
```

**防抖触发点**：所有可能的变更操作，包括：
- 字段编辑（表单/JSON 编辑器）
- 新增/删除节点
- 模板应用/修改
- 导入/新建 JSON
- 纯 UI 操作（展开/折叠树节点、切换 Tab、调整分隔条等）

**兜底定时器生命周期**：
- 页面加载 → 启动 60s 定时间隔
- 导出 JSON → 清除自动保存数据（数据已持久化到文件，不再需要自动保存快照）
- 页面关闭 → `beforeunload` 事件触发**立即保存**

### 变更检测

利用 `StoryStore._dataVersion` 自增标记：
- 保存时记录当前的 `_dataVersion`
- 兜底检查时对比 `_dataVersion` 是否变化
- 无变化则跳过保存，避免无用写入

### 启动恢复流程

```
main.js 启动
    │
    ├─→ 检查 storyeditor_document 是否存在
    │      │
    │      ├─ 不存在 → 正常启动，空状态
    │      │
    │      └─ 存在 → 弹出恢复确认对话框（ConfirmDialog）
    │                   │
    │                   ├─ 「恢复」→ 加载数据到 store，清除 storyeditor_document
    │                   │           显示 Toast "已恢复自动保存的数据"
    │                   │
    │                   └─ 「丢弃」→ 清除 storyeditor_document
    │                                空状态启动
```

### 状态指示器

在 ActivityBar 底部增加保存状态指示器，类似 VS Code 风格：

| 状态 | 显示 | 说明 |
|------|------|------|
| `idle` | 空白 | 无章节数据 |
| `saving` | 💾 图标 + "保存中…" | 正在写入 localStorage |
| `saved` | ✓ "已自动保存" + 时间 | 保存成功，几秒后渐隐为仅显示时间 |
| `error` | ⚠ "保存失败" | 写入异常 |

## 四、实现顺序

1. **localStorage 迁移** — 实现 `logic-migration.js`，改造 `logic-storyTypes.js` 中的读写函数
2. **自动保存核心** — 实现 `logic-autoSave.js`，含防抖、兜底、beforeunload
3. **Pinia 集成** — 在 `storyStore.js` 中新增 autoSaveStatus 状态
4. **恢复对话框** — main.js 中集成恢复流程，复用 ConfirmDialog
5. **状态指示器** — AutoSaveIndicator.vue 组件，引入 ActivityBar
6. **更新导出/导入配置** — settingsPanel 中导出/导入格式同步为新结构
7. **测试验证** — 导入 JSON → 编辑 → 等待保存 → 刷新页面 → 恢复
